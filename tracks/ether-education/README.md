# Track: Ether Education Protocol (EEP)

**Priority:** P1 — Near-Term  
**Status:** Design Phase  
**Lead:** Seeking Volunteer

---

## Purpose

The Ether Education Protocol delivers free, censorship-resistant education through the Noble Beacon sovereign stack. It rejects the credential-industrial complex in favour of knowledge that strengthens life, sharpens reason, and builds resilient communities.

## Core Principles

- **Free Forever**: No tuition, no subscriptions, no paywalls. Knowledge is a commons.
- **Offline-First**: All lessons work without internet via Lumen mesh and Vibe Pods.
- **Anti-Credential**: Learning is verified by demonstration, not by certificates from institutions.
- **Culturally Sovereign**: Communities adapt content to their language, context, and needs.

## The Five Strands

| Strand | Domain | Core Question |
|--------|--------|---------------|
| **Reason** | Logic, mathematics, critical thinking | How do I know what is true? |
| **Harmony** | Music, rhythm, pattern recognition, sacred geometry | How do I perceive the hidden order? |
| **Literacy** | Rhetoric, persuasion, anti-manipulation, language | How do I communicate without deceiving? |
| **Memory** | History, cultural heritage, spaced repetition | How do I honour what came before? |
| **Resilience** | Craft, repair, agriculture, first aid, survival | How do I sustain life when systems fail? |

## Lesson Architecture

Each lesson follows the **Seed → Root → Branch → Fruit** progression:

```
┌─────────────────────────────────────────────────┐
│  SEED: The Question                              │
│  A single provocative question that opens the   │
│  learner's curiosity. Never a statement.        │
├─────────────────────────────────────────────────┤
│  ROOT: The Foundation                            │
│  Core concept, historical context, first        │
│  principles. Maximum 500 words.                 │
├─────────────────────────────────────────────────┤
│  BRANCH: The Exploration                         │
│  Practical exercises, thought experiments,      │
│  connections to other strands.                  │
├─────────────────────────────────────────────────┤
│  FRUIT: The Demonstration                        │
│  A concrete task that proves understanding.     │
│  Peer-verified, not machine-graded.             │
└─────────────────────────────────────────────────┘
```

## Delivery Mechanisms

| Channel | Bandwidth | Use Case |
|---------|-----------|----------|
| Lumen Mesh | Low (text, compressed audio) | Remote communities, offline |
| Vibe Pod | Medium (full lessons, images) | Individual study |
| Browser Extension | High (multimedia, interactive) | Connected learners |
| Physical Print | Zero bandwidth | Archival, disaster resilience |

## Content Guidelines

- **No propaganda**: Present multiple perspectives. Let the learner decide.
- **No jargon without definition**: Every technical term must be explained in plain language.
- **No artificial difficulty**: Complexity should serve understanding, not gatekeeping.
- **Source everything**: Every factual claim links to its source in the archive.
- **Include the counter-argument**: For every position, present the strongest objection.

## Quality Standards

Every lesson must pass:
- **The Three Horizons Test**: Does it help today, tomorrow, and the unknown future?
- **The Ward Checkpoint**: Does it create hidden power or dependency?
- **The Plain Language Test**: Can a 14-year-old understand the core concept?
- **The Source Test**: Is every factual claim traceable to a verified source?

## Open Questions
